# The Lab

The Lab is an audit space for evaluating incoming ideas, tools, Reels/posts, workflow opportunities, and data-source/scraper experiments.

It is designed to keep early intake lightweight while giving promising ideas a clear path into incubation, existing projects, or archive.

## Start Here

Read `MANIFEST.md` first when trying to find relevant workflows, handoffs, tests, decisions, routing rules, or naming rules without knowing exact filenames.

Use `README.md` for the high-level project model and `MANIFEST.md` as the repo discovery index.

## Core Zones

### Intake

Raw incoming material and first-pass audits.

Use for:

- Reels, posts, videos, screenshots, transcripts, links
- New tools or workflow claims
- Shopping, lifestyle, education, prompting, business, or workflow ideas
- Quick triage before investing in deeper research

Default chat name:

`Intake — Reels, Tools, Ideas`

### Incubator

Validated ideas and tool tests that have passed initial usefulness proof but do not yet belong to an existing project.

Use for:

- Small experiments
- Tool comparisons
- Workflow prototypes
- Business or process ideas that need more maturity before project handoff

Default chat name:

`Incubator — Validated Ideas + Tool Tests`

### Framework

System design for The Lab itself.

Use for:

- Routing rules
- Naming schemes
- Project architecture
- Audit rubrics
- Thresholds for moving ideas between zones

Default chat name:

`Framework — Audit System Design`

### Archive

Historical context and mixed prior work.

Use for:

- Old mixed chats
- Prior test records
- Historical decision context

Default chat name:

`Archive — Audit Lab v0 / Mixed Chat`

## Routing Stages

Use this lightweight flow:

1. Inbox
2. Triage
3. Evidence Pass
4. Audit
5. Route

## Routing Outcomes

An item may be routed to:

- Reject / Archive
- Pattern Bank
- Watchlist
- Tool Stack
- Incubator
- Existing Project Handoff
- Dedicated Experiment

## GitHub Logging Rules

Use this repository, `Unl0ckedzERO/The-Lab`, as the durable ledger for sanitized workflows, tool tests, decisions, routing rules, naming rules, and project handoff notes.

Do not commit:

- raw datasets
- signed/private links
- API keys
- credentials
- sensitive metadata
- unredacted API responses

Summarize and sanitize before logging.

## Operating Principle

Intake should stay low-friction and somewhat messy. Mature ideas should not stay in intake forever.

When an idea proves useful and belongs to an existing project, create a handoff and move implementation there.

When an idea is promising but has no project home yet, move it to Incubator.
